# Demo 场景完整版

以下 4 个场景用于演示 Engagement Plan skill 的完整闭环流程（EP → Call Plan / Executive Briefing → PMR）。

> ⚠️ **Disclaimer:** 场景中的公司名称为真实企业，但**所有人物姓名、对话内容、态度描述、引用语句均为虚构**，仅用于演示 skill 的输出格式和质量标准。不代表任何真实商业关系或个人观点。

---

## 场景 1：Anker（制造/消费电子）— Internal Executive Briefing

- **背景：** AWS VP 下周拜访 Anker 深圳总部，需要内部准备会对齐议程、分工和 key messages
- **类型：** Internal Executive Briefing Meeting
- **路径：** EP → Executive Briefing → PMR
- **关键人物（内部）：** Account SA, Account Manager, Solutions Architect Lead
- **💬 用户开启：**
  > "我们 VP 下周要去 Anker 深圳，我得把内部的 prep 材料弄出来，你帮我理一下我们这边要对齐哪些东西？"

📁 [anker-internal-eb/](./anker-internal-eb/)

---

## 场景 2：理想汽车（汽车/智能制造）— 第 1 次拜访

- **背景：** 智能驾驶数据湖 + OTA 平台，多云评估阶段，首次正式会面
- **类型：** Call Plan（新客户冷启动）
- **路径：** EP → Call Plan → PMR
- **关键人物：** 智能驾驶VP, 平台架构总监
- **💬 用户开启：**
  > "我刚接手了理想汽车这个客户，下周二第一次去见他们智能驾驶的 VP，你觉得我应该聊点啥、怎么准备？"

📁 [li-auto-first-visit/](./li-auto-first-visit/)

## 场景 3：SHEIN（电商/快时尚）— 第 5 次拜访

- **背景：** 全球 CDN 优化 + 推荐系统 + GDPR 合规，深度合作阶段推进方案落地
- **类型：** Call Plan（老客户深度推进）
- **路径：** EP → Call Plan → PMR
- **关键人物：** CTO, Head of Infrastructure
- **💬 用户开启：**
  > "SHEIN 下周三又要见了，这次主要想把 GDPR 那块往前推一步，帮我想想这次会议怎么安排比较好？"

📁 [shein-deep-engagement/](./shein-deep-engagement/)

---

## 场景 4：港交所 / HKEX（金融）— Executive Briefing（客户来访 EBC）

- **背景：** 港交所 CTO 团队下月来访 AWS 办公室，需要准备完整的 Executive Briefing 文档
- **类型：** Executive Briefing（客户 EBC）
- **路径：** EP → Executive Briefing → PMR
- **关键人物：** HKEX CTO, Head of Cloud Strategy; 内部：Account Manager, GCR Leadership
- **💬 用户开启：**
  > "我下周有一个港交所的 EBC，帮我准备一下。"
